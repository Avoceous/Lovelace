Lovelace is a supple toolkit for building hybrid mobile app UI.

This is an example of a fully responsive content you can make. Take note that this is a hybrid app or a web based app using only html, css and javascript.

How it became android/ios app?. Great question, html & css cannot be loaded directly into a mobile device, it needs some framework, in this case we used Apache Cordova as a wrapper for all the contents including javascript. It has a web view which means what you see in a normal browser is the same as what you've seen in this screen.

Why Apache Cordova? why not using Ionic, React Native or even the most popular framework nowadays like Flutter? Wonderful question simply it's because Apache Cordova is a mature framework and still maintained by the cordova community, but what matters the most is its learning curve where you can utilize your web development skills and learn everything from it from a short period of time. By using Apache Cordova you will be able to create a mobile app within a few minutes because the documentation is highly organized and precised. Since it's a cross platform framework you can also build a desktop and a windows phone app.

How about its functionalites? can it perform like a native android java app or a native ios swift app? That's a terrific question. We will not lie to ourselves hybrid mobile app is quite different from a native mobile app it has pros & cons, hybrid app may result in a slower performance specially when its not fully optimized, unlike a native app like java or swift its speed is absolutely fast because it uses heavy optimzation, the source code will be compiled into a native code by using C/C++ programming language for example. But Apache Cordova can be your best friend it helps you understand how a native app works because community developers build their own javascript native plugins to be use in a cordova app such as accessing device camera, file manager, location, status bar, splash screen and a lot more which can make your Cordova app close enough to a native mobile application. Overall its up to the developer how he/she manage the code to make it even more cleaner and make the app smoother when its already performing. The app you're going to build depends on your creativity as well as a supple/flexible code will always matters because it also helps your hybrid app run smoother again the lighter the app the faster it can perform or in other words less is more & less is power.

In summary I built Lovelace to make your productivity even more easier so that you can save a lot of time re-inventing the wheel such as a basic app bar, drawer, floating action button, bottom navigation bar even a night mode feature to your app. I also installed some plugins for you, such as splash screen, status bar, social sharing, dialogs, network information if you want your app to run only when there's an internet and even in app browser plugin if you want to convert your website into a mobile app. You just have to do some simple configuration and remove something which you think unnecessary to you.

How can I create my very own android/ios app using Lovelace? Simple,

First open your terminal if you're using windows then use cmd and paste this command, git clone https://github.com/softDev28/Lovelace and click enter.

Second create your first cordova app and navigate the cordova app you created.

Third, Delete the www folder and replace it with the www folder located in the Lovelace folder.

Fourth, open config.xml located in the Lovelace folder and copy the code starting from the name tag until plugin tag, don't include the widget tag.

Fifth if you don't have Apache Cordova installed in your computer make sure to open Lovelace folder and open guide.txt for further instruction about the installation of Apache Cordova Framework.

If you're all set just run the command cordova platform add android/ios to tell the framework which platform you want to achieve.

Lastly the compilation process, just run this command cordova run, build or emulate android.

Its time to test the app now in your own phone and publish it in google playstore or in the app store.
